# Журнал изменений

Запись формата «что / зачем / результат» обязательна для каждого PR, меняющего код: без неё робот `doc-sync` краснит и мердж заблокирован. Форточка для заведомо тривиальной правки — метка PR `skip-changelog`.

## 2026-07-22
- Подключён конвейер разработки: журнал + коммит-gate (`doc-sync`), GPT-ревьюер (`ai-review`), правила для агентов (`AGENTS.md`), защита `main`. Зачем: изменения не должны попадать в прод без следа и независимого ревью. Результат: работа только через PR по единому процессу (канон — `OneBusinessCorportate/.github/dev-process.md`).
