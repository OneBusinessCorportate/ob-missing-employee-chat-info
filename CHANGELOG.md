# Журнал изменений

Запись формата «что / зачем / результат» обязательна для каждого PR, меняющего код: без неё робот `doc-sync` краснит и мердж заблокирован. Форточка для заведомо тривиальной правки — метка PR `skip-changelog`.

## 2026-07-24
- Подсказки к цифрам на дашбордах: единый источник пояснений `public/metric-info.js` и всплывающие тултипы (значок ⓘ) на «Проблемных чатах» и «Контроле клиентов» — по каждому разделу, столбцу, блоку-цифре и строке (что за таблица, как считается число, какие формулы, что такое «негативные» и как они исключаются). Зачем: без пояснений непонятно, откуда берутся цифры, и нельзя задать точечные вопросы. Результат: рядом с каждой цифрой доступно объяснение; текст можно поддерживать в одном файле.
- Правило синхронности подсказок: `AGENTS.md` + новая проверка `tooltips` в `doc-sync` — при изменении логики расчёта (`lib/problemChats.js` / `lib/clientChecks.js` / `lib/clientCount.js`) в том же PR обязано обновиться `public/metric-info.js` (форточка — метка `skip-tooltips`). Зачем: подсказка должна всегда совпадать с формулой. Результат: пояснения обновляются при каждом пуше, иначе мердж заблокирован.

- Доступность подсказок: значок ⓘ внутри плиток-кнопок (фильтры/статистика) теперь неинтерактивный (без `role`/`tabindex`) — это валидный HTML (у `<button>` не может быть фокусируемых потомков); в заголовках и ячейках таблиц значок остаётся полноценным контролом с доступом с клавиатуры. У всех значков добавлен нативный `title`-фолбэк. Зачем: замечание кросс-модельного ревью о недопустимых вложенных интерактивных элементах. Результат: разметка валидна, клавиатурная навигация и наведение работают.
- Синхронизация правила подсказок: проверка `tooltips` в `doc-sync` теперь охватывает и определения вью в `sql/*.sql` (не только `lib/*.js`), а формулировка правила в `AGENTS.md`/`README.md` уточнена под реально проверяемые пути. Зачем: замечание ревью — правило заявляло вью Supabase, а gate их не покрывал. Результат: enforcement совпадает с документацией.
